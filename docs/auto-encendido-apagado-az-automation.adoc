== Introducción

En este tutorial se indica cómo configurar el autoencendio y autoapagado de una máquina virtual en Azure. Esto es útil para evitar que la máquina virtual esté encendida y consuma recursos innecesarios, lo que puede resultar en costos adicionales.

Se describe como realizarlo con Logic Apps (consumption). Logic Apps es un servicio de Azure que permite automatizar flujos de trabajo y procesos empresariales.

== Pasos con Logic Apps

Antes de empezar, consulta el *grupo de recursos* de la máquina virtual que quieres encender y apagar automáticamente, así como la *región* donde se encuentra. 

=== Crear la Logic app (consumption)

En el Portal Azure: _Logic Apps -> Create_


.Creacion de la Logic App (consumption)
image::automation/logic-app-create-consuption.png[role="thumb", align="center", width=60%]


Elige el *mismo grupo de recursos* y la  *misma región* que la VM, para que se pueda acceder a ella desde la Logic App. Debes darle un nombre, por ejemplo `start-jenkins`.


.Parámetros de la Logic App (consumption)
image::automation/logic-app-create-consuption2.png[role="thumb", align="center", width=60%]

=== Configurar el disparador

Una vez creada la Logic App, ve a _Development tools > Logic app designer_.  Crea un disparador (_Add trigger_) de tipo _Recurrence_, que permite ejecutar la Logic App en un horario específico, por ejemplo, todos los días a las 09:00 horas para encender la VM.

.Diseñador de Logic App: añadir disparador
image::automation/logic-app-create-consuption3.png[role="thumb", align="center", width=90%]

```
Interval: 1
Frequency: Week
Time zone: (UTC+01:00) Madrid
Days: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday
Hours: 9
Minutes: 0
```

.Parámetros del disparador _Recurrence_
image::automation/logic-app-recurrence-parameters.png[role="thumb", align="center", width=90%]


=== Añadir acción: Start VM

Haz clic en _Add an action_.

.Add new Action
image::automation/logic-app-add-an-action-png.png[role="thumb", align="center", width=90%]

Busca la acción _Start Virtual Machine_.

.Add new Action: Start Virtual Machine
image::automation/logic-app-add-an-action-start-vm.png[role="thumb", align="center", width=90%]


La acción requiere `Create a new connection`. Deja el tipo de conexión predeterminado (`OAuth`) y haz clic en _Sign in_

.Add new Action: Create a new connection
image::automation/logic-app-add-an-action-new-connection.png[role="thumb", align="center", width=90%]

Esto abrirá un panel donde podrás autenticarte con tu cuenta de Azure y permitir que la Logic App acceda a tus recursos.

.Add new Action: Connection authentication
image::automation/logic-app-add-an-action-create-connection.png[role="thumb", align="center", width=60%]

[NOTE]
====
Para la segunda Logic app (Stop VM), no es necesario crear una nueva conexión, ya que puedes reutilizar la conexión creada para la primera Logic App.
====

A continuación, selecciona la *suscripción* y el *grupo de recursos* donde se encuentra la VM, y luego selecciona la *Virtual Machine* que deseas encender.

.Parámetros del disparador
image::automation/logic-app-start-vm-parameters.png[role="thumb", align="center", width=90%]

[WARNING]
====
No es necesario guardar cambios, se van guardando automáticamente. Al finalizar haz clic en _Publish_ para publicar la Logic App y que empiece a ejecutarse según el horario configurado.
====

=== Probar la acción: Run history

Una vez configurada la acción, puedes probarla haciendo clic en _Run history_. Puedes ver el historial de ejecuciones. 

.Probar la accion: Run history
image::automation/logic-app-start-vm-run.png[role="thumb", align="center", width=90%]

[WARNING]
====
Solo quedaría verificar manualmente que la VM se ha encendido correctamente.
====


=== Nueva Logic App: Stop VM

Crea una nueva Logic app repitiendo los pasos para apagar la VM. En este caso, `Deallocate Virtual Machine`, se configura la hora de apagado de la VM, por ejemplo, *todos los días* a las *23:00* horas.

[NOTE]
====
La diferencia entre _Stop_ y _Deallocate_ es que _Stop_ detiene la VM pero sigue consumiendo recursos, mientras que _Deallocate_ libera los recursos asociados a la VM, lo que puede resultar en un ahorro de costos. Por lo tanto, se recomienda utilizar _Deallocate_ para apagar la VM automáticamente.
====

Como resultado deberás tener 2 Logic Apps configuradas, una para encender la VM y otra para apagarla.

.Listado de logic apps
image::automation/logic-app-list-created-start-stop.png[role="thumb", align="center", width=90%]


== Facturación de Logic Apps 

La facturación de Logic Apps se basa en el consumo, solo pagas por las acciones y ejecuciones que realizas.

Actualmente Microsoft Azure incluye de forma gratuita un número limitado de ejecuciones de Logic Apps por suscripción, que suele ser suficiente para la mayoría de los escenarios de automatización. Por ejemplo, para encender y apagar una máquina virtual una vez al día, el número de ejecuciones será muy bajo y, por lo tanto, no incurrirás en costos adicionales significativos.

== Conclusión

Has configurado con éxito el autoencendido y autoapagado de una máquina virtual en Azure utilizando Logic Apps. Esta solución te permite optimizar el uso de recursos y reducir costos al asegurarte de que la VM solo esté activa cuando sea necesario.

De hecho, el ahorro obtenido por apagar diariamente la VM suele ser muchísimo mayor que cualquier coste potencial de Logic Apps, por lo que es una solución muy rentable para gestionar el encendido y apagado automático de máquinas virtuales en Azure.
